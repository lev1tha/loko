# Loko ERP — Express + Business

Комплексная ERP-система для логистической компании **Loko**. Два направления:

* **Loko Express** — доставка карго Китай → Кыргызстан (учёт товаров, продаж, маржи).
* **Loko Business** — мультивалютные закупки: сом → юань → оплата китайским поставщикам.

Код модульный (отдельные Django-приложения + сгруппированный фронтенд), направления
расширяются независимо.

## Стек

| Слой | Технологии |
|------|------------|
| Бэкенд | Django 4.2 LTS · DRF · SimpleJWT |
| Фронтенд | React 18 · Vite · React Router · Axios |
| Дизайн | дизайн-система Cal.com (через `getdesign` → `DESIGN.md`) |
| БД | SQLite (dev), легко заменить на PostgreSQL |

## Структура

```
backend/
  loko/        # настройки, корневой роутинг API
  accounts/    # кастомный User + роли (Админ / Кассир), JWT, команда seed
  finance/     # Account (мультивалюта), Expense (+статьи OpEx), Transfer (+конвертация),
               # AppSettings, движок отчётов reports.py (ООПИУ / ОДДС)
  express/     # Sale: цена клиента + динамическая себестоимость + маржа
  business/    # Deposit (депозиты), Debt (кредиторская/дебиторская)
frontend/src/
  pages/       # Login, Dashboard, Sales, Expenses, Accounts, Transfers,
               # Deposits, Debts, Reports, Settings, Users
  components/  # Layout (адаптивный сайдбар с секциями), UI-примитивы, иконки
```

## Запуск

> ⚠️ На этой машине порт **8000** занят сторонним процессом, поэтому бэкенд настроен
> на порт **8009** (`frontend/.env` → `VITE_BACKEND_URL`). Если у вас 8000 свободен —
> запускайте на 8000 и поправьте одну строку в `frontend/.env`.

### Бэкенд

```bash
cd backend
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py seed                     # пользователи, 3 счёта Express, 6 счетов Business, настройки
python manage.py runserver 127.0.0.1:8009
```

Демо-логины: `admin / admin123` (Администратор), `kassir / kassir123` (Кассир/Менеджер).

### Фронтенд

```bash
cd frontend
npm install
npm run dev                               # http://localhost:5173
```

### Импорт реальных данных из Excel

```bash
python manage.py import_business          # «Локо Бизнес 2.0.xlsx» (01–23.06.2026)
python manage.py import_express --path "<путь к Финансовый_учет_Локо…xlsx>"   # 2223 прихода + 163 расхода
```

Команды загружают реальные операции в БД и сходятся с контрольными итогами файлов **до копейки**:

| Направление | Выручка | Себестоимость | Чистая прибыль | Итоговый остаток |
|---|---|---|---|---|
| Business | 520 605 | 475 304.30 | 34 820.70 | 926 265.83 |
| Express  | 4 421 060.67 | — | — (OpEx 3 404 420.69) | — |

Отчёты **ОПиУ/ОДДС** имеют фильтр **Направление** (Всё / Express / Business), чтобы сверять с каждым
исходным Excel. Налог на прибыль — **4%** (настраивается). Курс юаня — 13.1. Нераспознанные оплаты
(«Банк не указан», «Смешанная», «Не оплачено») попадают на отдельный счёт «Банк не указан».

## Loko Express

* **Код клиента** — главный идентификатор (номер/код).
* **Два режима суммы** (как в реальном учёте):
  * *По весу* — `Вес × 3$ × 90 сом` + динамическая себестоимость (база **150 сом/кг**, `0.50 кг → 75 сом`).
  * *Прямая сумма* — сумма начисления вводится вручную (вес необязателен).
* **Начисление vs оплата** (раздельный учёт):
  * `Сумма начисления` → выручка **ОПиУ**, по «дате операции».
  * `Сумма оплаты` → приток **ОДДС** и баланс счёта, по «дате оплаты».
  * Разница = **дебиторка** (по продажам) / **кредиторка** (по расходам), видна во вкладке «Задолженности».
* Параметры цены снимаются на момент продажи; маржа считается автоматически.

### Соответствие реальным данным (Excel)
Модель приведена в соответствие с реальным журналом Локо: «Сумма начисления»/«Сумма оплаты»,
две даты (операции/оплаты), методы оплаты по счетам, **количество операций** (приходных/расходных —
в отчётах и на дашборде), расширенные **ОПиУ** (Валовая маржа %, Операционная прибыль, Прочие
доходы/расходы, Чистая рентабельность %) и **ОДДС** (остаток на начало/конец, операционная/финансовая
секции, «Свод оплат» по счетам).

## Loko Business

* **6 мультивалютных счетов:** Вичат, Алипей, ICBC, ICBC 0687, 9148 (CNY) и Мбанк (KGS).
* **Мультивалютный баланс** с тумблером отображения **сом / юань**.
* **Конвертация:** покупка юаня из Мбанка по **ручному курсу** при каждой операции
  (списание KGS → зачисление CNY).
* **Депозиты:** принятые депозиты **не** становятся выручкой автоматически — только по команде
  «Признать как выручку». Отправка поставщику → расход «Оплата поставщикам».
* **Задолженности:** кредиторская и дебиторская в отдельной вкладке.

## Финансовый модуль

Расход требует выбора категории: **OpEx**, **Оплата поставщикам**, **Неоперационная (Другое)**,
**Изъятие собственника**. Для OpEx обязательна статья:

* Аренда · ФОТ · Подоходный налог · Соц.фонд · **Прочие расходы** (комментарий **обязателен**).

### ООПИУ (P&L) — строгая многоуровневая формула

```
Выручка               = Продажи Express + Признанные депозиты Business
Валовая прибыль       = Выручка − Себестоимость
Прибыль до налога     = Валовая − Операционные расходы (Аренда+ФОТ+Подоходный+Соцфонд+Прочие)
Чистая прибыль        = До налога − Налог            (ставка из настроек или из фильтра отчёта)
```

ОДДС (Cash Flow) — реальный приток/отток, включая депозиты, поставщиков, изъятия собственника.
Все суммы консолидируются в сомы (юань — по курсу из настроек). Оба отчёта фильтруются по
**периоду** и **виду оплаты** (нал/безнал).

## API (кратко)

| Эндпоинт | Назначение |
|----------|------------|
| `POST /api/auth/login/` · `/auth/refresh/` | JWT-вход / обновление (login — единственный публичный) |
| `/api/sales/` (+ `summary/`, `quote/`) | продажи Express |
| `/api/expenses/` | расходы (категория + статья OpEx) |
| `/api/transfers/` | переводы и конвертация валют |
| `/api/accounts/?module=EXPRESS\|BUSINESS` | счета |
| `/api/deposits/` (+ `recognize/`, `send-to-supplier/`) | депозиты Business |
| `/api/debts/` (+ `close/`) | задолженности |
| `/api/settings/` | цена, себестоимость, курс юаня, ставка налога (PATCH — админ) |
| `/api/reports/pnl/` `?tax_rate=` | ООПИУ за период |
| `/api/reports/cashflow/` · `/balances/` · `/debts/` | ОДДС, остатки, задолженности |

## Адаптивность

От **360px** (сайдбар → выезжающее меню, один столбец) до **4K** (контент ограничивается,
сетки в несколько колонок). Flexbox/Grid, `clamp()`-типографика, относительные единицы,
векторные SVG-иконки.
